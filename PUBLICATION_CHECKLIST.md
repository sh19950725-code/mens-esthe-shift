# シフト管理システム 公開前チェックリスト

## 1. ローカル確認

- [ ] `npm run build` がエラーなしで完了する
- [ ] ログイン・ログアウトができる
- [ ] キャストを登録・編集できる
- [ ] シフトを登録・編集・削除できる
- [ ] 部屋の重複警告が動作する
- [ ] スマホ幅で文字やボタンが切れない

## 2. Supabase

- [ ] `casts`、`rooms`、`shifts`、`profiles` テーブルが存在する
- [ ] 各テーブルでRLSが有効になっている
- [ ] 未ログイン状態ではデータを取得できない
- [ ] 一般スタッフが管理者権限を変更できない
- [ ] `service_role` キーをフロントエンドで使用していない

## 3. 環境変数

公開先へ次の2項目を設定します。

```env
NEXT_PUBLIC_SUPABASE_URL=SupabaseのProject URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=SupabaseのPublishable key
```

注意事項：

- `.env.local` をGitHubへアップロードしない
- `service_role` キーを設定しない
- キーの前後へ余計な空白や引用符を付けない

## 4. Supabase Auth

- [ ] 公開URLをSupabaseのSite URLへ登録する
- [ ] パスワード再設定のRedirect URLへ公開URLを追加する
- [ ] 実際のスタッフ用アカウントでログインを確認する
- [ ] 退職したスタッフのアカウントを無効化する運用を決める

## 5. 公開後

- [ ] 公開URLをスマホで開ける
- [ ] ホーム画面へ追加できる
- [ ] オフライン表示が動作する
- [ ] CSVをダウンロードできる
- [ ] 印刷画面を開ける
- [ ] システム診断ですべて正常になる

## 6. 定期運用

- [ ] 月1回、Supabaseのバックアップ状況を確認する
- [ ] 不要なスタッフアカウントを削除または無効化する
- [ ] 操作履歴を定期的に確認する
- [ ] 機能追加前にGitでコミットする
